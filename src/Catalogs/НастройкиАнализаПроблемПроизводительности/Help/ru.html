Справочник содержит настройки анализа проблем производительности, необходимые для определения проблемных ключевых операций.

Анализ осуществляется регламентным заданием. С результатами анализа можно ознакомиться в регистре сведений "Результат автоматического анализа производительности".

Анализ использует механизм, аналогичный отчету "Анализ производительности (центр мониторинга)".

В результате анализа формируется список ключевых операций, имеющих проблемы производительности, отсортированный по возрастанию важности и APDEX. К каждой ключевой операции указывается конфигурация, в которой она зарегистрирована. Для регистрации берется последняя релизная версия, по которой есть достаточное количество данных. Можно регулировать объем зарегистрированных ошибок с помощью реквизитов "минимальное значение APDEX", "Регистрировать первые" и "Максимальный приоритет". Также на количество регистрируемых ошибок оказывает влияние настройки профиля ключевых операций, в табличной части которого указывается минимальное количество замеров, необходимых для анализа каждой ключевой операции.

Справочник имеет следующие реквизиты:

  • Профиль ключевых операций - ссылка на справочник "Профили ключевых операций". Определяет список ключевых операций, для которых выполняется анализ, а также параметры ключевой операции: целевое время, важность, минимальное количество замеров.
  • Минимальное значение APDEX - указывает значение ADPEX, ниже которого ключевая операция считается проблемной с точки зрения производительности.
  • Глубина интервала анализа, в днях - определяет интервал, на котором будет производиться анализ производительности. Анализ всегда осуществляется на начало текущего дня.
  • Регистрировать первые - определяет, сколько ключевых операций, из проанализированных, будет записано в регистр сведений "Результат автоматического анализа производительности".
  • Максимальный приоритет - устанавливает отбор на проанализированные ключевые операции по важности. Ключевые операции с важностью меньше либо равной указанному значению будут записаны в регистр сведений "Результат автоматического анализа производительности".

На вкладке "Конфигурации" указывается список конфигураций, для которых необходимо проводить анализ.

На вкладке "Статусы информационных баз" указывается список статусов информационных баз. Если информационная база имеет статус, отличный от указанного, то ее данные в анализе не участвуют. Если список пуст - отбор не работает.
