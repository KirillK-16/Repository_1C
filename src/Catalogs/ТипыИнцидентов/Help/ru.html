Тип инцидента - это классификация инцидентов по месту возникновения и характеру сообщения.

Если инциденты создаются на основании показателей, то каждая карточка "Обнаружение инцидентов" со своими условиями отбора показателей формирует отдельный типа инцидента.

Если инциденты создаются по результатам работы контрольных процедур, каждая контрольная процедура ассоциируется с собственным создаваемым типом инцидента.

Внешние источники инцидентов тоже экспонируют набор типов инцидентов, к которым затем привязываются открываемые инциденты.

Тип инцидента содержит список тегов. В случае внешних источников теги может определить прикладное решение, определяющее тип и администратор ЦКК может по своему желанию дополнить теги локальными. В случае внутренних инцидентов ЦКК можно оперировать только локальными тегами.

Для типа можно указать ответственного по умолчанию: зону ответственности или конкретного пользователя. В случае указания зоны ответственности все пользователя, входящие в зону, будут иметь возможность посмотреть инциденты типа как "свои".

По умолчанию инциденты на панели инцидентов сортируются по уровню и числу срабатываний. Но можно на уровне каждого типа переопределить положение инцидента, подняв какие-то записи вверх (они помечаются красным цветом) или наоборот опустив их в списке вниз (они помечаются зеленым цветом). Это делается с помощью поля Порядок. Чем больше указанное в нем значение (по умолчанию - 0), тем выше инцидент в списке на панели инцидентов. Положительные числа окрашиваются красным цветом, отрицательные - зеленым.

Если тип инцидента создан автоматически на основании контрольной процедуры или показателя ЦКК, поле "Объект ЦКК" содержит ссылку на исходный объект, который является родителем типа инцидента. В случае типов, добавленных внешними источниками поле "Объект ЦКК" пустое.

Пользователь ЦКК может для типа инцидента зафиксировать метод реагирования на него. Это можно сделать, добавив запись во вкладку "Методы реагирования". Знания можно зафиксировать с помощью написания статьи в карточке или прикрепив файл. Для одного типа может существовать более одного метода реагирования. Информация по методам реагирования накапливается и отображается для каждого экземпляра инцидента с этим типом.
