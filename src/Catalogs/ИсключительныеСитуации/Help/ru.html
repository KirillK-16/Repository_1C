Исключительные ситуации

Справочник исключительных ситуаций предназначен для группировки и учета записей технологического журнала. На его основе могут быть построены статистические отчеты.

Для исключительной ситуации указываются следующие реквизиты:

Даты создания и изменения - заполняются системой автоматически

Наименование - изначально заполняется системой автоматически по тексту исключения платформы

Вид - ссылка на справочник Виды исключительных ситуаций. Содержит текущую резолюцию по ошибке. Изначально присваивается значение "Не разобрано". В дальнейшем, при обработке ошибки, вид можно менять, например на "Зарегистрировать в системе контроля ошибок", "Штатное поведение", "Исправлено" и пр., а затем использовать для фильтрации ошибок в отчетах

Важность - важность ошибки. Может использоваться для фильтрации в отчетах

Код ошибки - произвольная текстовая строка для хранения регистрационного номера ошибки в системе контроля ошибок

Описание - произвольный текст, подробно описывающий ошибку, заполняется пользователем во время ее разбора

Для каждой исключительной ситуации указывается список условий, которым должна удовлетворять строка технологического журнала для того, чтобы быть отнесенной к данному исключению. При импорте данных технологического журнала в рамках контрольной процедуры "Мониторинг системных ошибок" производится анализ строк журнала. Те строки, которые имеют типом EXCP и удовлетворяют хотя бы одному из условий, приписываются к соответствующей исключительной ситуации. Если при импорте не найдено ни одной подходящей записи - создается новая запись в справочнике.

Условия в свою очередь состоят из нескольких критериев. Критерий содержит условие сравнения и значение, с которым будет сравниваться поле.Чтобы удовлетворять условию, строка журнала регистрации должна удовлетворять всем критериям.

При редактировании условий исключительных ситуаций происходит автоматическое переназначение строк технологического журнала. Строки, которые больше не удовлетворяют критериям, проходят стандартную обработку, проводимую при первичном импорте технологического журнала - ищутся или создаются условия для их группировки. Если же в результате изменения критериев некоторые строки журнала , ранее отнесенные к другим исключениям, начинают удовлетворять редактируемому, выводится форма запроса. Пользователь может отметить те исключительные ситуации, записи которых должны быть перенесены в редактируемое. При этом соответствующие записи справочника будут удалены, а строки техжурнала перенесены. Если же такой перенос не был запланирован (условие было изменено по ошибке), можно полностью или частично отказаться от переноса, сняв флажки у соответствующих строк на форме.
