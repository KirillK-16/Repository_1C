Экземпляры сценариев представляют собой задания на выполнение сценариев администрирования и выполняют автоматические действия по администрированию машин, заданные в сценариях.

Работу с экземплярами сценариев можно вести через два интерфейса:

·         Форма списка справочника «Экземпляры сценариев»

 

·         Календарь задач администрирования

  

Форма списка справочника Экземпляры сценариев

Форма списка позволяет:

·         добавить новый экземпляр сценария;

·         скопировать существующий сценарий (через мастер копирования);

·         стартовать выполнение сценария;

·         стартовать тест сценария;

·         запланировать сценарий;

·         прервать сценарий;

·         продолжить выполнение неуспешного сценария (продолжение выполнения с повтором неуспешного шага);

·         вернуть сценарий к начальному состоянию;

·         сформировать чек-лист выполнения по выбранным сценариям.

 

 

Действия по управлению экземпляром доступны также на форме элемента.

 

Запуск экземпляров сценария (задач) администрирования

Заполнение экземпляра сценария

Для заполнения нового экземпляра сценария необходимо:

 

1.      Выбрать контур администрирования.

2.      Выбрать сценарий, на основании которого будет заполнен экземпляр.

Будет создано дерево экземпляра сценария аналогичное дереву сценария, заполнены шаблоны параметров и параметры из сценария.

3.      Значение реквизита «Режим выполнения» будет унаследовано из выбранного сценария. В экземпляре можно его переопределить при необходимости.

4.      Установить соответствие реальных единиц оборудования (Агентов) шаблонам оборудования.

 

5.      В экземпляре должны быть дозаполнены обязательные параметры шагов и, при необходимости, переопределены дополнительные параметры. Также в экземпляре можно добавить шаблоны заполнения или переопределить значения существующих.

 

 

6.      При необходимости в экземпляре можно указать комментарий.

Дополнительные возможности

1. Отложенное выполнение шагов

Для сокращения времени недоступности, при этом сохранив сценарий целостным, есть возможность запустить сценарий и выполнить «подготовительные» шаги сразу, а собственно начало недоступности отложить на запланированное время. Для этого в дереве сценария есть колонка «Стартовать не ранее...». В ней задается время, не ранее которого должен стартовать данный шаг (и все последующие).

 

В примере на картинке шаги до установки блокировки базы выполнятся сразу после запуска сценария. Установка блокировки и фактическое начало недоступности произойдут в 20:00.

2. Отключение и включение шагов

В дереве экземпляра сценария можно включить отключенные по умолчанию в сценарии шаги или выключить включенные при необходимости. Это позволяет обеспечить гибкое использование сценариев, а также избавляет от необходимости создавать избыточное количество сценариев.

 

 

Также шаги можно по умолчанию выключить на уровне сценария. Тогда соответствующие шаги во всех экземплярах, создаваемых на основании такого сценария , будут также выключены по умолчанию.

3. Состояния экземпляра сценария

Сценарий может находиться в одном из следующих состояний:

·         активные:

o   Стартован – сценарий выполняется;

o   Тест стартован – сценарий запущен в режиме тестирования (не выполняет изменение параметров системы);

o   В процессе отката – в сценарии выполняются шаги отката;

·         неактивные:

o   Не запланирован – первичный статус для созданного экземпляра сценария. Такой экземпляр не включен в план, не отображается в календаре, не запланирован к выполнению;

o   Запланирован – сценарий включен в план запуска, будет запущен во время, указанное в реквизите «Плановая дата запуска»;

o   Тест завершен успешно;

o   Тест завершен неуспешно;

·         финальные:

o   Завершен успешно;

o   Завершен успешно (с оговорками) - есть неуспешно выполненные шаги в блоках «Вспомогательные шаги»;

o   Завершен неуспешно – сценарий завершился неуспешно, состояние системы не изменено;

o   Катастрофа! – сценарий выполнен неуспешно, состояние системы переведено в некорректное состояние, требуется вмешательство Администратора;

o   Прерван – сценарий прерван в процессе выполнения. Состояние контура неизвестно, требуется вмешательство Администратора.
