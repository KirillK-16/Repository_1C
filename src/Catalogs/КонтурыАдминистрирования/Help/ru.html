Специфика задач администрирования и автоматизации на разных внедрениях очень разнообразна и в определенный момент может возникнуть необходимость разделять задачи и конечные инструменты администрирования (скрипты) в разрезе различных команд, проектов, зон ответственности.

 

Для задач такого разделения в системе есть вспомогательный справочник «Контуры администрирования (проекты)». Также реквизит «Контур администрирования» является обязательным в большинстве основных объектов (экземпляры сценариев, задания планировщика и т.д.). Контур администрирования характеризует не набор машин, на которых могут выполняться сценарии администрирования, а набор скриптов (на уровне исполняющей части), а также сценариев и команд (на уровне управляющей конфигурации), которые будут использоваться для решения задач администрирования в разных зонах ответственности

Контур администрирования хранит информацию о пути установки дистрибутива скриптов исполняющей части и конфигурационных файлов к ним на конечной машине.

 

 

Есть ограничение, обоснованное тем, что Агент, исполняющий сценарии для разных Контуров администрирования один и оперирует одним путем установки дистрибутива исполняющей части центра администрирования. Ограничение в том что для разных контуров нельзя указать разные каталоги установки дистрибутива исполняющей части.

 

Разделение может осуществляться на уровне дистрибутива скриптов (по умолчанию каталог src, может быть разделен на несколько копий для каждого контура) и конфигурационных файлов к ним (по умолчанию каталоги win_configs/linux_configs, может быть разделен на несколько копий для каждого контура).
