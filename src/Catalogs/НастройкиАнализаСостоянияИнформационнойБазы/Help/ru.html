Справочник содержит настройки анализа, необходимые для определения статуса информационной базы.

Анализ осуществляется регламентным заданием. С результатами анализа можно ознакомиться в регистре сведений "Результат анализа информационных баз".

Анализ базируется на расчете коэффициента корреляции Пирсона, который может принимать значение от -1 до 1.

Анализ состоит из следующих компонентов:

1) Анализ активности пользователей.

Анализируется активность пользователей не указанную глубину (в неделях). Выбирается последняя полная неделя и сравнивается с каждой предшествующей. Для каждого сравнения рассчитывается коэффициент корреляции. Далее, в зависимости от метода оценки, выбирается или максимальный коэффициент, или рассчитывается средний коэффициент. Если он больше коэффициента, определенного в настройках, и активность пользователей больше минимальной, то результат анализа положительный.

2) Анализ изменения данных в разрезе объектов метаданных (анализируется прирост).

Анализируется изменение данных в разрезе объектов метаданных. Для каждого объекта метаданных выбирается информация об изменении количества данных. Для анализа желательно выбирать такие объекты метаданных, в которых количество данных с течением времени увеличивается линейно. В ходе анализа выполняется проверка, есть ли изменение количества данных больше, чем установленное значение "минимальный прирост". Если проверка выполняется, то рассчитывается коэффициент корреляции для количества данных и времени. Если используются весовые коэффициенты, то после анализа всех объектов метаданных рассчитывается общий коэффициент корреляции, который сравнивается с общим. Если весовые коэффициенты не используются, то сравнивается коэффициент корреляции объекта метаданных с установленным в настройке. Если коэффициент корреляции больше установленного в настройках значения, то результат анализа положительный.

3) Анализ первоначального идентификатора информационной базы в "Центре мониторинга" на соответствие уникальному идентификатору демонстрационной базы.

Сравнивается постоянный уникальный идентификатор информационной базе в "Центре мониторинга" с уникальным идентификатором демонстрационной базы, установленным на вкладке "Конфигурации".

На вкладке "Конфигурации" указывается список анализируемых конфигураций. Для каждой конфигурации можно указать уникальный идентификатор демонстрационной базы.

На вкладке "Прирост данных" указываются настройки анализа изменения (прироста) данных:

  • Глубина анализа данных, в месяцах - определяет, за какой период от текущей даты необходимо анализировать динамику изменения количества данных.
  • Использовать весовые коэффициенты - определяет, каким способом оценивать динамику изменения количества данных. Если галочка установлена, то для каждого объекта метаданных в табличной части необходимо указать весовой коэффициент. Сумма весовых коэффициентов всегда равна 1. Если галочка не установлена, то будет осуществляться сравнение рассчитанного коэффициента корреляции с тем, который установлен для объекта метаданных.
  • Общий коэффициент корреляции - используется при сравнении коэффициента, рассчитанного с помощью весов.

 

В табличной части указываются следующие параметры:

  • Объект метаданных - объект метаданных, для которого будет проведен анализ.
  • Минимальный прирост - увеличение количества данных объекта метаданных, которое является минимально необходимым для признания поведения такого объекта характерным для продуктивной базы.
  • Коэффициент корреляции - используется при отключенных весовых коэффициентах. Если рассчитанный коэффициент корреляции больше указанного значения, то анализ покажет положительный результат.
  • Весовой коэффициент - используется для расчета общего коэффициента корреляции для изменения указанных объектов метаданных.

 

На вкладке "Активность пользователей" указываются настройки анализа активности пользователей:

  • Коэффициент корреляции пользовательской активности - является целевым коэффициентом корреляции. Используется при сравнении с общим коэффициентом или с коэффициентом корреляции конкретных недель.
  • Глубина анализа активности, недель - определяет, за какой период необходимо анализировать данные. Из анализа исключаются недели, в которых были праздничные дни и перенос праздничных дней.
  • Метод анализа активности - выбирается вариант анализа. Если "искать максимальный коэффициент корреляции", то выбирается такое сравнение недель, где коэффициент максимальный. Если "рассчитывать средний коэффициент корреляции", то рассчитывается средний коэффициент по всем неделям, попавшим в анализ. Полученный коэффициент корреляции сравнивается с коэффициентом корреляции пользовательской активности.
  • Минимальная пользовательская активность. Значение сравнивается с максимальным количеством пользователей в последней полной неделе. Если пользователей было меньше этого значения, то независимо от коэффициента корреляции, результат такого анализа будет отрицательным.
