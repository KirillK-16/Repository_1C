Оценка производительности

APDEX выполнения ключевых операций.

Показатель позволяет вычислять зависимость от времени коэффициента APDEX выполнения ключевых операций в информационных базах, зарегистрированных в ЦКК.

Для работы показателя требуется, чтобы была запущена и настроена контрольная процедура "Контроль производительности".

Форма настройки показателя позволяет настроить

  • фильтр по информационной базе;
  • фильтр по пользователям, выполняющим ключевую операцию.

При настройке показателя нужно указать конкретную ключевую операцию, время выполнения которой будет использоваться при рассчёте APDEX, или выбрать "Общую производительность", и тогда APDEX будет вычисляться по всем ключевым операциям.

Также можно настроить параметры визуализации замеров на диаграмме на форме "Мониторинг":

  • Настройки вида линии серии на диаграммах (толщина, цвет, тип).
  • Настройки масштаба, используемого при отображении линии серии на диаграммах.
  • Параметры сглаживания замеров (поддерживается сглаживание методами скользящего среднего, линейного аппроксимирования или средствами однослойного персептрона).

 

Среднее время худших замеров.

Методика расчета

  1. Показатель имеет два настраиваемых параметра: "Период оценки", "Количество худших замеров".
  2. Выбираются N последних замеров по указанным ключевым операциям объекта контроля.
  3. Из выбранных N замеров выбираются худшие записи.
  4. Для выбранных худших записей считается среднее время.

ВНИМАНИЕ!!!

Среднее время худших замеров нескольких объектов вычисляется как сумма средних времен худших замеров каждого объекта, деленная на количество объектов.

Пример:

Будет рассчитано среднеее времени худших операций по двум выбранным объектам контроля: "Информационная база 02" и "Информационная база 03". Для объекта контроля "Информационная база 02" выбрано три ключевых показателя, для объекта контроля "Информационная база 03" выбрано пять ключевых показателей. Для объекта контроля "Информационая база 02" будет выбрано N = 100 последних замеров указанных ключевых показателей. Из данных замеров будет выбрано 25 худших замеров и для них посчитано среднее время. Предположим, что при расчете по объекту "Информационная база 02" мы получили среднее время худших замеров равное 1 секунде. Аналогично расчитали среднее время для объекта контроля "Информационная база 03" и получили среднее время худших замеров равное 4 секунды. Тогда среднее время худших замеров по выбранным объектам контроля "Информационная база 02" и "Информационная база 03" будет рассчитано: (1 + 1)/2 = 2,5 секунды.

Количество операций превышения целевого времени.

Методика расчета

  1. Показатель имеет два настраиваемых параметра: "Период оценки", "Порог превышения целевого времени".
  2. Выбираются N последних замеров по указанным ключевым операциям объекта контроля.
  3. Из выбранных N замеров выбираются записи, время которых превышает целевое время в указанное количество раз.
  4. Рассчитывается количество записей, время которых превышает целевое время.

ВНИМАНИЕ!!!

Количество операций превышения целевого времени нескольких объектов контроля вычисляется как сумма количеств операций превышения целевого времени каждого объекта.

Пример:

Будет рассчитано количество операций превышения целевого времени по двум выбранным объектам контроля "Информационная база 02" и "Информационная база 03". Для объекта контроля "Информационная база 02" выбрано два ключевых показателя, для объекта контроля "Информационная база 03" выбрано пять ключевых показателей. Для объекта контроля "Информационная база 02" будет выбрано N = 100 последних замеров указанных ключевых показателей. Из данных замеров будут выбраны записи, время замеров которых превышает целевое время в два раза. Предположим, что для объекта контроля "Информационная база 02" количество таких записей равно 12. Аналогично рассчитали количество операций превышения целевого времени для объекта "Информационная база 03" и получили значение 7. Тогда количество операций превышения целевого времени по выбранным объектам контроля "Информационная база 02" и "Информационная база 03" будет рассчитано: 12 + 7 = 19 операций.

Процент пользователей превышения целевого времени.

Методика расчета

  1. Показатель имеет два настраиваемых параметра: "Период оценки", "Порог превышения целевого времени".
  2. Выбираются N последних замеров по указанным ключевым операциям объекта контроля.
  3. Для  каждой записи из N последних замеров расчитывается признак превышения: 0 - нет превышения целевого времени, 1 - есть превышение целевого времени.
  4. Группируем выборку из полученных записей по пользователям, суммируем по признаку превышения.
  5. Рассчитываем процент записей со значением признака превышения целевого времени больше нуля к общему количеству записей.

ВНИМАНИЕ!!!

Процент пользователей превышения целевого времени нескольких объектов вычисляется как сумма процентов пользователей превышения целевого времени каждого объекта, деленная на количество объектов.

Пример:

Будет рассчитан процент пользователей превышения целевого времеи по двум выбранным объектам контроля "Информационная база 02" и "Информационная база 03". Для объекта контроля "Информационная база 02" выбрано два ключевых показателя, для объекта контроля "Информационная база 03" выбрано пять ключевых показателей. Для объекта контроля "Информационная база 02" будет выбрано N = 100 последних замеров указанных ключевых показателей. Предположим, что после анализа данных записей и группировки по пользователям получим следующую выборку:

Процент пользователей изменения целевого времени.

Методика расчета

  1. Показатель имеет четыре настраиваемых параметра
    1. "Период оценки" - разница в днях между текущим и предыдущим периодами выборки замеров целевого времени
    2. "Глубина выборки" - количество дней выборки замеров целевого времени для каждого периода
    3. "Дельта среднего времени замера" - изменение среднего целевого времени пользователя в текущем периоде по отношению к предыдущему в процентах
    4. "Минимальное количество замеров" - минимальное количество замеров пользователя за период для попадания в выборку
  2. Рассчитывается среднее время замеров пользователя по выбранным ключевым показателям за текущий период
  3. Рассчитывается среднее время замеров пользователя по выбранным ключевым операциям за предыдущий период
  4. Рассчитывается дельта изменения среднего времени замера пользователя текущего периода по отношению к предыдущему
  5. Рассчитывается процент пользователей, у которых дельта изменения среднего времени больше указанного в параметре "Дельта среднего времени замера"

ВНИМАНИЕ!!!

Процент пользователей изменения целевого времени нескольких объектов вычисляется как сумма процентов пользователей изменения целевого времени каждого объекта, деленная на количество объектов.

Пример:

Будет рассчитан процент пользователей изменения целевого времени по двум выбранным объектам "Информационная база 02" и "Информационная база 03". Для объекта контроля "Информационная база 02" выбрано два ключевых показателя, для объекта контроля "Информационная база 03" выбрано пять ключевых показателей.

Предположим, что мы рассчитываем значение показателя на момент времени равному 12ч 27мин 30.05.2014.

Рассчитываем среднее время замеров, количество замеров по пользователям за текущий период: где начало периода равно интересующий нас момент времени минус глубина выборки. Т.е. текущий период равен: с 12ч 27 мин 23.05.2014 по 12ч 27 мин 30.05.2014
