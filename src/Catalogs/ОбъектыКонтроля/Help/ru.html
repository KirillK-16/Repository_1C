Общие принципы работы

Компоненты системы, качество работы которых контролируется и поддерживается при помощи контрольных процедур, называются объектами контроля. ЦКК позволяет контролировать качество работы объектов следующих видов:

  • информационные базы,
  • кластеры «1С:Предприятия»,
  • компьютеры,
  • другие центры контроля качества;

Первое, что должен сделать пользователь при начале работы с ЦКК: описать все программные и аппаратные компоненты системы. Необходимо зарегистрировать в ЦКК все программные и аппаратные компоненты, которые принимают участие в работе системы.

Cледует описать: 

  • Все рабочие серверы, которые участвуют в работе системы. В частности: сервер СУБД, рабочие серверы 1С:Предприятия, терминальные сервера, сервер WEB и т.п.;
  • Кластеры 1С:Предприятия;
  • Информационные базы, участвующие в работе системы.

Такой подход к контролю качества позволит отслеживать любые проблемы качества работы контролируемой системы, но не позволит диагностировать инфраструктурные проблемы – такие как отключение электричества или сбой всей сети. Поэтому желательно иметь еще один - «внешний» ЦКК, который будет находиться полностью вне инфраструктуры контролируемой системы – на другой территории, с другим аппаратным и сетевым обеспечением. Этот «внешний» ЦКК может контролировать работоспособность внутреннего ЦКК, периодически получая от него сообщения по электронной почте.

Для «внешнего» ЦКК следует описать:

  • все центры контроля качества, которые необходимо контролировать при помощи данного «внешнего» ЦКК.

При настройке объекта контроля необходимо указать зону ответственности, которую будут разделять пользователи, отвечающие за настройку контрольных процедур данного объекта контроля. Пользователи, разделяющую эту зону ответственности, будут заниматься первоначальной настройкой контрольных процедур, а также исправлением всех критических ситуаций, препятствующих нормальной работе контрольных процедур.

Основная форма ЦКК

Основная форма ЦКК отображает перечень всех аппаратных и программных компонентов системы. При начале работы системы перечень содержит по одному предопределенному объекту каждого вида. Для каждого предопределённого объекта формируется задача по первоначальной настройке. После выполнения задачи ЦКК автоматически добавит необходимые контрольные процедуры и задачи по их настройке.
