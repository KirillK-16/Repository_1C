Мониторинг системных ошибок

Контрольная процедура позволяет вести автоматический контроль ошибочных ситуаций в работе "1С:Предприятие".
Контроль выполняется посредством автоматических настройки технологического журнала и его разбора. 
 
Перед тем как настраивать контрольную процедуру, необходимо в настройках рабочего сервера, являющегося объектом контроля для контрольной процедуры, указать каталог, в котором хранится файл logcfg.log.
 
При корректной настройке параметров рабочего сервера, большинство настроек контрольной процедуры заполняются автоматически рекомендуемыми значениями.
Внимание! Сетевое имя каталога технологического журнала и локальное имя каталога технологического журнала должны указывать на один и тот же каталог!
Сетевое имя будет использоваться для доступа к файлам технологического журнала при автоматическом разборе, а локальное имя требуется для настройки файла logcfg.xml.
 
 
Рекомендуемый порядок настройки контрольной процедуры
 
1. Настроить расписание, рекомендуемое значение: каждый день; каждые 600 сек.
2. Настроить режим оповещения о восстановлении.
3. Настроить режим оповещения о сбое и добавить задачи, при сбое которых будет производиться оповещение.
4. Оставить без изменения сетевое имя каталога технологического журнала (должны были корректно указать при настройке рабочего сервера).
5. Оставить без изменения локальное имя каталога технологического журнала на рабочем сервере (должны были корректно указать при настройке рабочего сервера).
6. Оставить без изменения локальный каталог временных файлов (автоматически создается во временном каталоге пользователя на сервере, под которым работает сервер 1С:Предприятие). При желании можно указать любой другой локальный каталог. Необходим доступ на чтение, запись, удаление.
7. Оставить без изменения через сколько часов информация будет удаляться из технологического журнала. Указанное значение будет использоваться как атрибут history элемента log при настройке технологического журнала. Рекомендуемое значение - 24 часа.
8. Оставить без изменения как долго информация, собранная контрольной процедурой будет хранится в информационной базе ЦКК. К выбору этого параметра следует подходить очень осторожно: если выбрать слишком большое значение, размер информационной базы ЦКК через некоторое время и некоторое количество разобранных технологических журналов станет неприлично велик. Рекомендуется для контрольной процедуры "Мониторинг системных ошибок" хранить собранные данные не более 480 часов.
