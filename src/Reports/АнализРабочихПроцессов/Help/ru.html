Отчет "Информация о перезапусках рабочих процессов".

Для работы отчета необходимо, чтобы для кластеров была настроена и запущена контрольная процедура "Контроль потребления памяти".

Отчет предназначен для представления данных по перезапускам рабочих процессов за определенный период. Полученные данные можно использовать для анализа перезапусков рабочих процессов кластера, которые могут быть следствием:

  • Утечек оперативной памяти вследствие ошибок в прикладном коде или платформе;
  • Некорректных настроек свойств кластера или рабочего сервера.

Информация представлена в разрезе кластеров и рабочих серверов.

 

Информация о полях отчета.

  • Поле "Кластер" показывает кластер серверов 1С:Предприятия;
  • Поле "Сервер" показывает рабочий сервер;
  • Поле "Время начала" показывает время старта рабочего процесса;
  • Поле "Время завершения" показывает время завершения рабочего процесса. Если рабочий процесс не заврешен то в поле будет отображаться значение "Выполняется";
  • Поле "Память, Mb" показывает количество оперативной памяти, потребляемой процессом в момент завершения. Если рабочий процесс не заврешен то будет отображено текущее значение потребляемой оперативной памяти;
  • Поле "Количество сеансов в момент завершения" показывает количество сеансов в момент завершения, которые были в клиент-серверном вызове. Если рабочий процесс не заврешен то будет отображено текущее количество сеансов в клиент-серверном вызове;
  • Поле "Процесс (Pid)" показывает Process ID рабочего процесса.

Параметры отчета.

  • Параметр "Период" задает период анализируемой информации;
  • Параметр "Только текущие процессы" позволяет отобрать в отчете только выполняемые в текущее время процессы;
  • Параметр "Отбор" позволяет задать отбор, например по отдельному кластеру.
